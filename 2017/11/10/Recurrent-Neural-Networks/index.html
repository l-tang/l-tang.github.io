Recurrent Neural Networks

RNN
Publish Date:   2017-11-10

Recurrent Neural Networks (RNNs) can be used for prediction. It can analyze time series data such as stock prices and tell you when you should buy/sell. It is also possible to predict the trajectory of the vehicle in an automatic driving system, thereby avoiding traffic accidents. Unlike many other neural networks, the length of input sequence for RNNs is flexible. In this way, you can replace RNNs with different parameter sequences, such as an article, or an audio sample, and this is one of the reasons why RNNs often excel in the NLP line. More importantly, RNNs have the ability to predict continuous data, so it can cope with innovative tasks including automatic arrangement, text writing and so on.

1. Recurrent Neurons

For most neural networks, the propagation direction is basically one direction, that is, the more common feed-forward network. For RNNs, its propagation direction is bidirectional. The simplest example is to receive input $x_\left(t\right)$ from only one recurrent neuron, and the output $y_\left(t-1\right)$ of the neuron at the last moment. If we extend it in timeline, we can get the effect shown in Figure 1-1.

Therefore, it is not very complex to generate a layer composed of recurrent neurons – just requires that every neuron receives $x_\left(t\right)$ and $y_\left(t-1\right)$ as the input at any time. For each recurrent neuron, there are two sets of weights: $w_x$ and $w_y$ for the input $x_\left(t\right)$ and $y_\left(t-1\right)$, so the output $y_\left(t\right)$ can be represented as:

$$y_\left(t\right)=\phi\left(x^T_t \cdot w_x + y^T_\left(t-1\right) \cdot w_y + b\right)$$

where $b$ is the bias term and $\Phi$ is the activation function (many researchers prefer to use hyperbolic tangent rather than ReLU in RNNs), and thus the output of the whole layer is:

$$Y_\left(t\right)=\phi\left(X_\left(t\right)\cdot W_x + Y_\left(t-1\right)\cdot W_y + b\right)$$

simplified to:

$$Y_\left(t\right)=\phi\left([X_\left(t\right)Y_\left(t-1\right)]\cdot W + b\right) \ with \ W = \begin{bmatrix}W_x\\W_y\end{bmatrix}$$

  • $Y_\left(t\right)$ is a matrix containing all outputs at time step $t$ with size $m\times n_{neurons}$, where $m$ is the number of instances in this mini-batch and $n_{neurons}$ is the number of neurons.
  • $X_\left(t\right)$ is an $m\times n_{inputs}$ matrix containing all input instances, where $n_{inputs}$ is the number of input features.
  • $W_x$ and $W_y$ are matrices containing connection weights for $X_\left(t\right)$ and $Y_\left(t-1\right)$. Since the size of $W_x$ and $W_y$ are $n_{inputs}\times n_{neurons}$ and $n_{neurons}\times n_{neurons}$, $W_x$ and $W_y$ are usually concatenated as a single weight matrix $W$ of shape $\left(n_{inputs} + n_{neurons}\right)\times n_{neurons}$.
  • $b$ is a vector containing all bias terms for every neuron.

2. Memory Cells and The Input/Output Sequence of RNNs

Because recurrent neuron is concerning of all inputs at last time step $t-1$ while it is dealing with the output at the time step $t$, recurrent neuron can be treated as a unit with memory, and the part of the network, being able to preserve the state of sequence, is called a memory cell, or simply a cell. The most basic format of a memory cell is one recurrent neuron or one layer consisting of recurrent neurons.

Generally, we use $h_\left(t\right)$ to stand for the hidden state of a memory cell at time step $t$. The value of $h_\left(t\right)$ is apropos the input at the current time step and the hidden state at the last time step, which can be described as: $h_{(t)}=f(h_{(t-1)}, x_{(t)})$. The output at the time step $t$, denoted $y_{(t)}$, is also determined by the previous hidden state and the current inputs. $h_{(t)}$ and $y_{(t)}$ are not always equal in some convoluted cases. The application of RNNs is very flexible. An RNN can generate an output sequence by simulating an input sequence, namely Seq2Seq. Thus it can be used to predict time series, such as stock prices. Another application method is to input a sequence, and then ignore other outputs, and keep only the last output. This method can be used to vectorize a sequence, that is, Seq2Vec. For example, a movie-related comment can be turned into a word sequence, and then input this sequence into a network and expect an output as the score of this movie. The opposite way is to keep only the first input and set all the inputs at other times to 0, and then let the network output the entire sequence, so it is a Vec2Seq network. For example, you can enter an image and then output a text description about it. The last common method is that you can use a Seq2Vec network as the encoder,and then connect a Vec2Seq network as the decoder, which forms a delayed Seq2Seq network. For example, you can translate one language into another. First, language A is input into the encoder section and converted into a vector value. Then use the decoder to decode this vector and convert it to language B. This two-step model is called the Encoder-Decoder model. Compared with ordinary Seq2Seq, this way does not need to wait for the entire sentence to be fully input before translating.

4. Train RNNs

Generally speaking, training RNNs needs to be implemented using backpropagation through time (BPTT). In the BP process, the loss function calculates the return gradient of the output sequence, and then returns the gradient BP to the entire network. Even if the output sequence selects only a part of all the outputs, the gradient calculated by the loss function will BP the entire network.

Just like the ordinary BP process, first it is a forward propagation through an unrolled network, and then the loss sequence is used to evaluate the output sequence. Then, the loss gradient is back-propagated and finally the model parameters are updated through the BPTT gradient. Taking the MNIST data set as an example, if each picture is taken as a 28 * 28 sequence, it can be classified using RNN. We set 150 neurons, and add an FC layer containing 10 neurons after the output at the last moment, so that each neuron corresponds to an output. Finally, a softmax layer is used to achieve the final multi-classification.

5. Deep RNNs

Like other types of networks, RNNs can also implement deep RNNs by implementing more layers of cells. It is very convenient to implement deep RNN in TensorFlow. You only need to create multiple cells, and then integrate independent cells together through the MultiRNNCell method.

The state at this time is a tuple that contains a tensor for each layer. The tensor represents the final state of the cells in this layer. If you set state_is_tuple = False when using MultiRNNCell, state will concatenate the states of the three layers to become a tensor.
